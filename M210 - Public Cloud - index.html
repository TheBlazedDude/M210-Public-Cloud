M210 - Public Cloud für Anwendungen nutzen

Definition

Eine Public Cloud ist ein Bereitstellungsmodell bei dem sich Computing-Ressourcen im Besitz von Anbietern befinden, die diese Ressourcen verwalten und für mehrere Mandanten über das Internet bereitstellen (und somit als gemeinsame Ressource genutzt wird).

was zeichnet eine Public Cloud aus

  1. Ressourcenzuweisung: Die Mandanten außerhalb der Firewall des Anbieters greifen gemeinsam auf Cloud-Services und virtuelle Ressourcen zu, die über dessen Infrastruktur, Plattformen und Softwareanwendungen bereitgestellt werden.
  2. Nutzungsvereinbarungen: Ressourcen werden bedarfsabhängig verteilt, das heißt aber nicht, dass wir es hier mit Pay-as-you-go-Modellen zu tun haben. Manche Kunden, wie die wenigen Forschungseinrichtungen, die die Massachusetts Open Cloud verwenden, nutzen Public Clouds kostenfrei.
  3. Management: Die Mindestleistungen des Anbieters bestehen in der Wartung der Hardware für die Cloud, dem Netzwerksupport sowie der Verwaltung der Virtualisierungssoftware und eventuell das erstellen von automatisierte Self-Service-Funktionen

Wer kann Public Clouds erstellen

Im Prinzip jeder, weshalb es weltweit Tausende von Anbietern gibt. Aber zu den größten – und beliebtesten – zählen heute Alibaba Cloud, Amazon Web Services (AWS), Google Cloud, IBM Cloud und Microsoft Azure.

Funktionen von Public Clouds

  • Skalierbarkeit: Die Cloud-Ressourcen können bei Bedarf schnell und einfach skaliert werden.
  • Selbstbedienung und On-Demand: Dienste können in Echtzeit bereitgestellt werden, oft über ein Selbstbedienungsportal.
  • Zahlung nach Verbrauch: Statt fester Kosten zahlen Nutzer nur für die Ressourcen, die sie tatsächlich verbrauchen.
  • Zuverlässigkeit: Public Clouds bieten eine hohe Verfügbarkeit und Redundanz durch den Einsatz mehrerer Rechenzentren.
  • Wartung und Updates: Der Cloud-Anbieter kümmert sich um die Wartung, einschließlich Hardware-Austausch, Software-Updates und Sicherheitspatches.
  • Globale Reichweite: Nutzer können Dienste von überall auf der Welt zugreifen und dabei auf regionale Datenzentren des Anbieters zurückgreifen.
  • Integration und Interoperabilität: Viele Public Clouds bieten Integration mit bestehenden Systemen und unterstützen Standards, die den Datenaustausch und die Interoperabilität erleichtern.
  • Sicherheit und Compliance: Anbieter implementieren Sicherheitsmaßnahmen und Compliance-Protokolle, um Daten zu schützen und regulatorischen Anforderungen zu genügen.
  • Breite Palette an Diensten: Public Clouds bieten eine Vielzahl an Diensten an, von einfachem Hosting bis hin zu komplexen Anwendungen wie Datenanalyse, künstliche Intelligenz und Internet der Dinge (IoT).
  • Automatisierung: Viele Aufgaben, die mit der Bereitstellung und dem Management von IT-Ressourcen verbunden sind, können automatisiert werden, um Effizienz und Konsistenz zu erhöhen.

Diese Funktionen machen die Public Cloud zu einer attraktiven Option für Unternehmen aller Größen, die ihre IT-Infrastruktur modernisieren und von den Vorteilen der Cloud-Technologie profitieren möchten.

Begriffe

  • FTP: File Transfer Protocol: Ist ein CLient-Server-basiertes Netzwerkprotokoll, mit dem sich Daten in einem TCP/IP-Netzwerk übertragen lassen (seit 1971, FTP ermöglicht es, Daten zwischen verschiedenen Rechnern auszutauschen).
  • Docker/Container:
  • Cloud-Lösung: wenn Daten, Rechenleistung oder Anwendersoftware nicht über eigene Hardwareressourcen abgearbeitet werden. Stattdessen werden die Ressourcen von Servern und Cloud-Anbietern verwendet, auf die zuvor die Daten über das Internet hochgeladen wurden.
  • Serverless: bezieht sich auf ein cloudnatives Entwicklungsmodell, bei dem Entwickler Anwendungen erstellen und ausführen können, ohne Server verwalten zu müssen. Natürlich sind die Server noch physisch vorhanden, nur wurden sie von der App-Entwicklung getrennt.
  • CI/CD Pipeline: auch Continuous Integration/Continuous Deployment-Pipeline genannt, ist eine Reihe von Automatisierungsschritten, die die Bereitstellung von Anwendungen vereinfachen und beschleunigen.

Wieso heisst eine Cloud eigentlich Cloud

eine Wolke (Cloud) besteht aus vielen Regentropfen. aber nur als Wolke sind sie sichtbar. Clouds im Internet wurden immer als solche Wolken gesehen.

wie wird eine Cloud definiert

  1. nicht physisch vorhanden bei mir
  2. nicht einzeln vorhanden
  3. verbund von gemeinsamen Ressourcen/Computerressourcen

Merkmale des Cloud-Computing mit Zugriff via NIST

  • on demand self service: Nutzung auf Abruf, bekomme als Kunde immer sofort Zugriff
  • broad network access:über Netzwerk (Internet) und Clients auf die Dienste zugreifen.
  • ressource pooling: Zusammenlegung von Ressourcen, Serverkapazitäten werden gebündelt und als einheitliche CLoud zur Verfügung gestellt. -> Economics of scale
  • rapide elasticity: Zur Verfügung stehende Ressourcen werden angepasst und es entsteht der Eindruck eines unbegrenzten Speicherplatzes.
  • measured service: Überwachung des Dienstes, die einzelnen Cloud-Server werden immer wieder überwacht und optimiert. -> nur was genutzt wird, kostet auch

Vorteile von Cloud-lösungen gegenüber on Premise-lösungen

  • Bezahlung nach Gebrauch
  • geringe Gesamtkosten (geringe Anschaffungskosten)
  • geringer Wartungsaufwand
  • weltweiter Zugang

Nachteile von Cloud-lösungen gegenüber on Premise-lösungen

  • Kosten (Bezahlung nach Gebrauch)
  • provider login wechseln
  • Daten sind nicht bei mir/weniger Kontrolle
  • Sehr hohe Komplexität (FTP war im Vergleich sehr einfach)
